📘 What is Hamming Code?

Hamming Code is a linear error-correcting code invented by Richard W. Hamming at Bell Labs in 1950. It was designed to automatically detect and correct single-bit errors that occur during data transmission or storage — a critical need in the era of punch-card computing where errors were common.

For every k data bits, Hamming adds r parity bits at positions that are powers of 2 (positions 1, 2, 4, 8...). Each parity bit covers a specific subset of bit positions determined by the binary representation of those positions, and the total codeword length becomes n = k + r.
The condition is: 2^r ≥ n + 1.

Example sizes in this simulator:

  • 4 data bits → 3 parity bits → 7-bit codeword (Hamming(7,4))
  • 8 data bits → 4 parity bits → 12-bit codeword (Hamming(12,8))

The parity bits are calculated so the XOR of all bits they cover equals 0 (even parity). At the receiver, recomputing parity checks produces a syndrome. If the syndrome is zero, no error occurred; otherwise, the syndrome value points to the exact erroneous bit position, which can be flipped to correct the error.

⚡ What Causes Errors?
Transmission errors occur due to electromagnetic interference, noise, signal attenuation, or hardware malfunctions. Hamming codes help detect and correct single-bit errors automatically.
